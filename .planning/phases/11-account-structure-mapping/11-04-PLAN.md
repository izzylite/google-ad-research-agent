## Anti-patterns

- **Don't run ad_group_match.py before Phase 8.** Without
  raw/google-ads-perf.json + raw/google-ads-search-terms.json, the script
  exits 0 with an empty mapping (skipped_reason='phase8_artifacts_absent'),
  which downstream is fine — but you've gained nothing. Run Phase 8 first.
- **Don't mutate ad-group-mapping.json by hand.** It's regenerated by
  ad_group_match.py each run. Edit ranked.json or brief.md upstream instead.
- **Don't normalize Unicode characters in existing ad-group names.** En-dash
  (U+2013) and em-dash (U+2014) are common in Google Ads account names. The
  whole pipeline preserves them byte-for-byte (Pitfall 2). Manual edits that
  strip them WILL break CSV imports.
- **Don't expect coverage > 50% in narrow verticals.** A 3-ad-group account
  vs. a 20-keyword ranked list often shows 30-40% coverage — that's correct
  behavior, not a bug. The Next Steps step 3 keeps the "Create ad groups"
  default; operator manually decides if existing names apply.
- **Don't pass --geo-focus values that include "County" suffix without
  spaces.** "PalmBeachCounty" fails the lookup; "Palm Beach County" works
  (the suffix is stripped during _build_city_filter normalization).

## Failure modes

- **report.md missing `## Geographic Focus` section even though brief has
  Geo focus:** brief.md `**Geo focus:**` line malformed. Verify with `grep
  '\*\*Geo focus:' {run_dir}/brief.md`. The regex requires `**` markers on
  both sides — alternative quotes (smart quotes, single asterisks) fail.
- **Next Steps step 3 still says "Create ad groups" despite high coverage:**
  Check `cat {run_dir}/ad-group-mapping.json | python -c "import json,sys;
  print(json.load(sys.stdin)['mapping_coverage_pct'])"`. If it's exactly
  50.0, that's correct — the threshold is strict > 50.0 (open-question 4).
- **positives.csv shows cluster slugs instead of existing ad-group names:**
  Coverage may be < 50% (low-confidence matches don't substitute). Check
  the mapping JSON; if a specific keyword expected to match shows
  `confidence: low`, inspect the `reason` field for jaccard score —
  similarity below 0.4 means the search-term token bag for that ad group
  doesn't overlap enough with the keyword.
- **ad_group_match.py crashes on ranked-enriched.json schema mismatch:**
  Phase 8 + Phase 9 must have run successfully. If ranked-enriched.json was
  manually edited, restore from `ranked.json` (the script falls back to that
  if ranked-enriched is absent or unparseable).

## Downstream contract

Phase 11 output that downstream tooling depends on:

- `{run_dir}/ad-group-mapping.json` — schema:
  - `matches[]` — list of `{keyword, existing_ad_group | null, confidence, score, reason}`
  - `unmapped_count` — count of confidence=low matches
  - `mapping_coverage_pct` — (high + medium) / total * 100.0 (Pitfall 7)
  - `computed_at` — ISO-Z UTC timestamp
  - `skipped_reason` — null on happy path; `"phase8_artifacts_absent"` on graceful skip
- `report.md` — `## Geographic Focus` section + conditional Next Steps step-3 rewrite
- `report.json` — top-level keys `geographic_focus` (always) + `ad_group_mapping_summary` (when mapping present)
- `export/positives.csv` — `Ad Group` column substituted for high/medium matches
- `export/ad_groups.csv` — existing ad-group names filtered out

This contract is stable from v1.2 forward; future v2 features must not break it.
