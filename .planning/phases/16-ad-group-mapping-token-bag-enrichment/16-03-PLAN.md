<objective>
**Goal:** Author RED tests that pin down the per-source max-Jaccard structural fix for ADGM-11. Tests must FAIL against the current full-union Jaccard implementation in `ad_group_match.py` and pass only after Plan 16-04 ships the algorithm change.

**Chosen structural fix (justification — per gap_summary recommendation + live e2e evidence):**

**Candidate #1: Per-source max-Jaccard.** Replace the current full-union scoring:

```
raw_j = _jaccard(kw_tokens, ag["token_bag"])  # current: full union
```

with:

```
name_j = _jaccard(kw_tokens, ag["name_tokens"])
crit_j = _jaccard(kw_tokens, ag["criterion_tokens"])
term_j = _jaccard(kw_tokens, ag["search_term_tokens"])
raw_j  = max(name_j, crit_j, term_j)          # new: per-source max
```

**Why this candidate over #2 (subsampling) or #3 (asymmetric similarity):**
1. **Live evidence drives it.** Plan 16-02 live e2e reason-field sample was `jaccard=0.10 (name=0.33 kw-criterion=0.00 search-term=0.00) intent_match=True`. Under per-source max, the same match would have scored 0.33 (clears the high threshold 0.30) instead of being diluted to 0.10. Directly addresses the observed failure mode.
2. **Smallest code delta.** Plans 16-00/16-01 already compute partial token sets (`name_tokens`, `criterion_tokens`, `search_term_tokens`) inside `_build_ad_group_index`; Plan 16-04 just rewires `build_mapping` to take the max instead of running Jaccard against the union bag. No new helpers, no new tuning surface.
3. **Preserves per-source attribution.** The reason-field format (`jaccard=X.XX (name=Y.YY kw-criterion=Z.ZZ search-term=W.WW) intent_match=B`) shipped in 16-01 already exposes per-source contributions — under max-Jaccard, the reason field becomes *causally* aligned with the score (the max'd source IS the score), making the format meaningful instead of decorative.
4. **Threshold space minimally perturbed.** Current `_THRESHOLDS = {0.30, 0.10}` are floor values; per-source max only shifts scores upward (since `max(a,b,c) >= jaccard(union)` whenever at least one source is non-trivially populated). Calibration risk is "do we tighten thresholds back toward Phase 11 0.7/0.4?" — a clean tightening, not a full retune (unlike asymmetric similarity which inverts the score distribution shape).
5. **Token-bag subsampling (#2) is excluded** because it introduces a new tuning surface (top-K per source) and can hide signal. Asymmetric similarity (#3) is excluded because it requires full recalibration and breaks the symmetric Jaccard intuition the rest of the codebase relies on.

**Purpose:** Lock in the test contract for Plan 16-04. RED tests must fail at HEAD (full-union Jaccard); pass once Plan 16-04 ships max-Jaccard.
