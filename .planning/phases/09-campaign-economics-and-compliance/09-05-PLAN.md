**Anti-patterns (do not do these):**
- Don't run Phase 9 before Phase 8. Bid logic needs `cpc_micros` from `ranked-enriched.json`.
- Don't hand-edit `references/compliance-verticals.json` Python-side — the file is operator-editable JSON.
- Don't mutate `ranked-enriched.json` outside `bid_suggest.py`. Additive-only contract.
- Don't write the Next-Steps checklist in Phase 9 (that's Phase 10 STEP-01).
- Don't manually format USD inside the three scripts. All math stays in micros; USD conversion happens only in render_report.py at the display boundary.

**Failure modes:**
- **Missing ranked-enriched.json:** Phase 8 must run first. Surface the actionable error.
- **Empty cluster CPC pool:** Some keywords land null + `no_cpc_data` flag. This is correct behavior. Report renders em-dash. Operator can override Max CPC manually in Editor.
- **Compliance false positive on neutral brief:** Operator can remove the offending token from `references/compliance-verticals.json`. The substring-with-word-boundary algorithm is conservative; rare false positives possible (e.g., "loaner" intentionally does NOT match "loan").
- **Unjoined keywords > 0 in forecast:** Cluster keyword string casing drifted from ranked. Cluster names typically come from Phase 4; if drift > 5%, re-cluster.

**Downstream contract (read by Phase 10):**
- `ranked-enriched.json[i]["suggested_max_cpc_micros"]` → Phase 10 fills Editor CSV `Max CPC` column (EXPT-01).
- `forecast.json["campaign_totals"]["daily_spend_mid_usd"]` → Phase 10 fills "Set daily budget to $X" in Next-Steps (STEP-02).
- `compliance-flags.json["matched_verticals"][i]["verification_url"]` → Phase 10 reorders Step 1 to "Complete {vertical} verification at {url}" when matched_verticals non-empty (CMPL-05 / STEP-01).
- `report.json["forecast"]` + `report.json["compliance"]` (CMPL-04) → downstream tooling can consume the same data without re-reading the sidecars.
