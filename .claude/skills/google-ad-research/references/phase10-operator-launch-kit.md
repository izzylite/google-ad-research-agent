# Phase 10 — Operator Launch Kit

Editor-importable CSV exports + bespoke Next Steps checklist. Last v1.1
phase. Operator copies CSVs into Google Ads Editor, follows the ordered
checklist, launches the campaign.

## When to run

After Phase 9 (`bid_suggest.py`, `forecast_budget.py`, `compliance_check.py`)
produces `ranked-enriched.json` (with `suggested_max_cpc_micros`),
`forecast.json`, and `compliance-flags.json`.

Phase 10 is **optional but high-value**:
- Three Editor-importable CSVs under `{run_dir}/export/` — paste-and-go
- Next Steps checklist substitutes brief + Phase 9 forecast values per run
- Compliance flag present → verification step promotes to step 1
- HTML report adds interactive checkboxes (localStorage per run)

## Prerequisites

- Phase 6 complete (`ranked.json`, `clusters.json`, `negatives.json`)
- Phase 8 complete (`ranked-enriched.json` with `cpc_micros`)
- Phase 9 complete (`forecast.json`, `compliance-flags.json`,
  `suggested_max_cpc_micros` field on ranked-enriched rows)
- No `.env` changes — pure compute

## Step 41: Confirm operator wants Phase 10

Ask:

> "Run Phase 10 (Operator Launch Kit)? Emits three Google Ads Editor
> v2.x CSVs to `{run_dir}/export/` and appends a bespoke Next Steps
> checklist to `report.md`. Pure compute — no API costs."

If no → stop. If yes → continue.

## Step 42: Write Editor CSVs

```bash
uv run "${CLAUDE_SKILL_DIR}/scripts/export_csv.py" --run-dir "{run_dir}"
```

Parse stdout JSON. Surface `positives_rows`, `negatives_rows`,
`ad_groups_rows`, `exports_dir`.

Exit codes:
- 0 → continue
- 2 → disk write error (PermissionError / OSError); offer retry
- 3 → missing input (ranked-enriched.json / clusters.json /
       negatives.json / brief.md); stop and surface which file

Produces:
- `{run_dir}/export/positives.csv` — Campaign, Ad Group, Keyword,
  Match Type, Max CPC, Final URL
- `{run_dir}/export/negatives.csv` — Campaign, Ad Group, Keyword,
  Match Type, Level (Strong→campaign, Considered/Investigate→ad_group)
- `{run_dir}/export/ad_groups.csv` — Campaign, Ad Group, Status,
  Default Max CPC

Byte contract: UTF-8 no BOM, CRLF line endings, RFC 4180 quoting.

## Step 43: Re-render report

```bash
uv run --with python-slugify --with python-dotenv --with tabulate \
  "${CLAUDE_SKILL_DIR}/scripts/render_report.py" --run-dir "{run_dir}"
```

`render_report.py` auto-detects the `export/` folder and emits:

- **Export Files** section (after Ranked Keywords) listing the 3 CSVs
  with relative paths
- **Next Steps** section (LAST section) — ordered 8-step ops checklist
  with brief + Phase 9 forecast substitution. If `compliance-flags.json`
  has matched verticals, a single combined verification step prepends
  to position 1, remaining steps renumber by list position
- HTML interactive Next Steps section with checkboxes; localStorage
  keys namespaced `gar_<slug>_step_<id>` (cross-run isolation)
- `report.json` gains top-level `exports[]` (CSV paths) and
  `next_steps[]` (ordered list of `{n, text, id}` dicts) keys

Surface to operator: report.md / report.json / report.html updated.
Phase 10 done — operator can import CSVs into Editor and follow checklist.

## Anti-patterns

- **Don't run `export_csv.py` without Phase 9 first.** Needs
  `suggested_max_cpc_micros` field on ranked-enriched rows; missing
  field means Max CPC column = "0.00" for every keyword.
- **Don't edit `export/*.csv` files manually then re-render.** The
  whole `export/` folder is regenerated by `export_csv.py`; manual
  edits are clobbered. Edit in Editor after import instead.
- **Don't trust Step 1 ordering when compliance is present.** CMPL-05
  promotes verification to step 1; the standard "Create campaign"
  becomes step 2. Operator should always read the rendered checklist,
  never assume position from documentation.
- **Don't change match-type case in CSVs.** `Phrase` / `Exact` / `Broad`
  title-case is the Editor v2.x spec; lowercase causes column-mapping
  errors on import.
- **Don't commit `{run_dir}/export/`.** Run folders are git-ignored;
  CSVs are local launch artifacts.

## Failure modes

- **Editor import error "Required column missing"**: usually a BOM in
  the CSV. `export_csv.py` uses `encoding="utf-8"` (not `utf-8-sig`) so
  this shouldn't happen — but check with `head -c 3 positives.csv` if
  doubt arises. BOM = `\xef\xbb\xbf` bytes.
- **Editor import error "Invalid match type"**: case sensitivity. Verify
  `Phrase` / `Exact` / `Broad` title-case in CSV.
- **Next Steps cluster names list shows `<clusters>`**: clusters.json
  was empty or missing. Phase 4 (clustering) failed silently; re-run
  it and re-render.
- **HTML checkbox state leaks across runs**: localStorage key not
  namespaced. Verify rendered HTML contains literal `gar_${slug}_step_`
  pattern in the renderNextSteps JS function.
